Folder Map & Usage Cheat Sheet

헷갈리는 기준은 하나다. app/plugin은 Codex가 읽는 플러그인 껍데기, app/harness는 실제 명령/검증 코드, projects/<slug>는 영상별 작업장, references/style은 여러 프로젝트에서 재사용하는 스타일 창고다.

app/plugin

  • 역할: Codex 플러그인 등록부. manifest, skills, agents, hooks가 들어간다.
  • 읽는 시점: 세션 시작, 관련 작업 요청, tool 사용 전후, Stop 시점.
  • 건드릴 때: 새 skill/agent/hook을 추가하거나 플러그인 정책을 바꿀 때만.
  • 중요 파일: .codex-plugin/plugin.json, skills/*/SKILL.md, agents/*/AGENT.md, hooks/hooks.json

app/harness

  • 역할: 운영 CLI와 Python 패키지. 프로젝트 생성, 자료 등록, QA, imagegen job, Seedance plan을 만든다.
  • 읽는 시점: uv run python scripts/*.py 명령을 실행할 때.
  • 건드릴 때: 폴더 규칙, 검증 로직, 게이트, RALPH, Seedance dry-run 동작을 바꿀 때.
  • 중요 파일: cli/*.py, package/safety_video_harness/*.py, schemas/*.json, templates/project/*

projects/<slug>

  • 역할: 한 편의 영상 프로젝트 전체 산출물. 입력 자료, 레퍼런스, 스토리, 이미지, 영상, QA 증거가 여기에 쌓인다.
  • 읽는 시점: 거의 모든 CLI가 --project projects/<slug>로 이 폴더를 기준 삼는다.
  • 건드릴 때: 이번 영상 자료를 넣거나 승인 이미지/레퍼런스/QA 파일을 관리할 때.
  • 핵심: 프로젝트별 레퍼런스는 여기에 들어간다. 전역 스타일 가이드와 섞지 않는다.

references/style

  • 역할: 여러 프로젝트에서 재사용하는 그림체/렌더링 스타일 카탈로그.
  • 읽는 시점: intake의 style_guide_id, extract_style_dna, 이미지 프롬프트 팀 단계.
  • 건드릴 때: 반복 사용 가능한 스타일 가이드를 추가할 때.
  • 중요 파일: catalog.json, <style-id>/STYLE_GUIDE.md, <style-id>/references/*

docs

  • 역할: 운영 설명서, 루브릭, few-shot, 로컬 참조 문서, 원페이저.
  • 읽는 시점: 평가, 프롬프트 작성, Higgsfield/Seedance 계획, 구조 파악 전.
  • 건드릴 때: 사용법이나 평가 기준이 바뀌었을 때.
  • 중요 파일: evaluation-rubrics.md, few-shot-examples.md, higgsfield-seedance-local-reference.md

Live Boundary

  • 원칙: live image/video/TTS는 승인 없이 실행하지 않는다.
  • 차단: pretooluse-live-veto.py가 paid/live성 실행을 막는다.
  • 허용: 먼저 --dry-run으로 job spec, 비용, 업로드 대상을 확인한다.
  • 증거: QA round, visual review, sampled frames를 남긴 뒤 다음 단계로 간다.

│   ├── candidates/                # 후보 보관함: 자동 prompt 반영 안 됨
│   └── approved/                  # 실제 prompt에 들어가는 승인 레퍼런스
│       ├── people/                # 포즈, 시선, 역할 행동
│       ├── work/                  # 작업상황, 위험상황, 예방행동
│       ├── spaces/                # 현장 구조, 동선, 통로, 위험구역
│       ├── style/                 # 프로젝트 한정 그림체/색감 참고
│       ├── camera/                # 구도, 앵글, 거리감
│       └── lighting/              # 조명, 날씨, 노출
├── sources/                       # 추출된 source facts, topics, citations
├── story/                         # scenes, storyboard QA, image prompt team plan
├── media/
│   ├── images/draft/              # 생성/수집된 후보 키프레임
│   ├── images/approved/           # 승인 키프레임, 덮어쓰기 금지
│   └── video/clips/               # Seedance/Higgsfield clip 결과
├── qa/                            # image/video manual review, visual contact sheet
├── gates/                         # Gate 1, Gate 2 승인 상태
├── project_config.json            # target seconds, style_guide_id, text delivery
└── HANDOFF.md                     # 다음 세션 인계 문서

레퍼런스 넣는 기준

  • refs/people: 같은 사람이 반복 등장해야 할 때.
  • refs/ppe: 안전모/조끼/장갑처럼 착용물이 중요할 때.
  • refs/equipment: 차량 축 수, 설비 형상, 제품 크기를 고정할 때.
  • refs/approved/*: 장면 구성이나 스타일처럼 prompt에 직접 반영할 승인 참고.

.md Sidecar 규칙

  • 이미지 옆에 같은 이름의 .md를 둔다.
  • 인물 다각도 이미지는 worker-001.profile.md처럼 공통 프로필을 둔다.
  • 내용은 “참고할 것 / 복제하지 말 것”만 짧게 쓴다.
  • 설명 없는 이미지는 파일명 기반 placeholder가 들어가므로 production용으로 약하다.

헷갈리면 이렇게

  • 모자/조끼/장갑: refs/ppe
  • 사람 자체: refs/people
  • 차량/장비: refs/equipment
  • 장면 배치: refs/approved/work 또는 spaces
  • 화면 틀/앵글: refs/approved/camera

Top-Level Flow

문서/상태 처리 AI/프롬프트 생성 QA/채점 승인/차단 산출물
01

인터뷰

자료, 주제, 길이, 이미지 밀도, 스타일, 레퍼런스, 승인 범위를 묻는다.

02

프로젝트 생성

init_project가 PLAN, AGENTS, HANDOFF, 기본 상태 파일을 만든다.

03

자료 등록

PPTX/SOP를 등록하고 media extract와 slide text extraction으로 근거 자료를 만든다.

04

주제 추출

topic-extractor가 교육 주제 후보와 source citation을 만든다.

05

스토리보드

story-writer와 storyteller가 source 기반 장면 계약을 작성하고 review dashboard를 만든다.

06

스토리 QA

rubric, few-shot, role evaluator, Arbiter로 Gate 1 후보를 검토한다.

G1

사용자 승인

스토리보드 승인 전 imagegen 또는 영상 단계로 넘어가지 않는다.

07

Asset Lock

cast, PPE, equipment, space/background, style reference를 먼저 고정한다. text-only 다중 컷은 draft exploration이다.

08

이미지 설계

lead-style, scene-prompt, arbiter가 asset lock 기반 imagegen/edit/compositing job spec을 만든다.

09

키프레임

Codex imagegen 또는 reference/edit chain 결과를 collect/record하여 images/draft에 저장하고 승인본은 images/approved로 이동한다.

10

이미지 RALPH

44/55, OpenCV MCP/휴리스틱/수동 시각 QA, identity/equipment/background/hazard-zone blocker를 검사한다. 실패 원인은 RALPH critique block으로 다음 프롬프트에 주입하고, 같은 blocker 3회 반복 시 upstream 수정으로 에스컬레이션한다.

G2

비용 승인

Seedance live는 비용 고지, upload 허용, Gate 2 승인 뒤에만 가능하다.

11

영상 QA/조립

start/end keyframes와 reference media pack 기반 Seedance 결과를 sampled frames, story-video alignment, subtitles, final assembly plan으로 검토한다.

실패 루프: 스토리보드 QA 실패는 storyboard 수정, 이미지 QA 실패는 RALPH 재생성 또는 upstream escalation, 영상 QA 실패는 유료 재생성 자동 실행 없이 propose_only 수정 제안만 남긴다.

Production Work Flowchart

작업 산출물 점수 평가 사용자 승인/차단 RALPH 개선 루프 Live/유료 경계

이 작업 흐름도는 운영 순서를 기준으로 한다. 스토리보드는 Gate 1에서 사용자가 직접 확인하고, 이미지는 RALPH 루프를 돌 수 있지만 영상은 비용 때문에 자동 루프를 돌지 않는다. production 키프레임은 asset lock, reference/edit chain, deterministic compositing 중 하나를 통과해야 하며 Seedance는 prompt-only가 아니라 start/end keyframes와 reference media pack을 lock layer로 사용한다.

Runtime Flowchart

Core scripts/state Skills/agents QA/Arbiter Hooks/Gates MCP/External tools

이 흐름도는 실제 실행 순서를 기준으로 한다. Hooks는 명령 전후에 끼어들어 차단하고, Skills/Agents는 산출물 작성과 평가를 담당하며, MCP/외부도구는 승인된 구간에서만 실행된다.

Plugin Manifest & Runtime Core

Manifest

.codex-plugin/plugin.json

  • 플러그인 이름: safety-video-harness
  • Codex에 노출되는 skill 6개를 등록한다.
  • 현재 운영은 로컬 repo에서 scripts와 docs를 함께 사용하는 방식이 가장 안정적이다.

Python Harness

safety_video_harness/

  • 프로젝트 상태, 게이트, 비용, QA, locks, evidence, imagegen job, Seedance plan을 관리한다.
  • qa_contract.py가 rubric/few-shot/artifact/blocker metadata를 표준화한다.
  • project_handoff.py가 프로젝트별 HANDOFF를 생성한다.

Agents

작동: evidence bundle 수집 -> 역할별 독립 review -> Arbiter decision -> approve/revise/regenerate/escalate -> llm-wiki에 라운드 기록.

storyteller

  • 역할: 선택 주제와 source facts로 스토리보드 초안을 만든다. citation 없는 안전 주장은 금지.
  • 구성: agents/storyteller/AGENT.md + references/storyteller-reference.md
  • 사용 예: “레미콘 충돌 예방 주제로 30초 스토리보드 초안을 만들어줘.”
  • 산출물: storyboard/scenes.json, 장면별 citation, 자막/오버레이 문구

lead-style-agent

  • 역할: 첫 키프레임을 기준으로 인물, 차량, 공간, 카메라, 렌더링 규칙을 style bible로 잠근다.
  • 구성: agents/lead-style-agent/AGENT.md + style-guides/<id>/STYLE_GUIDE.md
  • 사용 예: “sc01을 기준으로 이후 장면의 인물/PPE/차량/공간 bible을 만들어줘.”
  • 산출물: style, character, vehicle, space, camera bible

scene-prompt-agent

  • 역할: 각 장면을 이미지 생성 가능한 상세 prompt brief로 바꾼다.
  • 구성: agents/scene-prompt-agent/AGENT.md + 이전 QA blocker + 승인 reference
  • 사용 예: “sc03 장면을 이전/다음 컷과 이어지는 image prompt brief로 만들어줘.”
  • 산출물: previous continuity, current action, next setup, gaze targets, must preserve/avoid

visual-director-arbiter

  • 역할: 병렬 scene prompt 결과를 통합하고, 생성 전 일관성 문제를 차단한다.
  • 구성: agents/visual-director-arbiter/AGENT.md + prompt team plan + storyboard citations
  • 사용 예: “전체 이미지 프롬프트 세트가 한 영상처럼 이어지는지 generation 전 판단해줘.”
  • 산출물: ready_for_generation, revise_scene_prompt, revise_style_bible, revise_storyboard

continuity-qa

  • 역할: 스토리보드, 프롬프트, 이미지가 하나의 예방 흐름인지 점수화한다.
  • 구성: agents/continuity-qa/AGENT.md + docs/evaluation-rubrics.md
  • 사용 예: “승인 이미지 7장이 슬라이딩 체인으로 자연스럽게 이어지는지 검토해줘.”
  • 산출물: story flow score, sliding-chain 판단, identity/PPE/equipment blocker, regeneration delta

visual-continuity-director

  • 역할: 인물 등장/퇴장, 시선 동기, 동선, 위험구역, 교육 명확성을 영상 전후로 감독한다.
  • 구성: agents/visual-continuity-director/AGENT.md + sampled frames + approved images
  • 사용 예: “이 영상에서 사람이 갑자기 사라지거나 시선이 어색한지 봐줘.”
  • 산출물: qa/video_manual_review.json findings, blocker, frame evidence

video-qa

  • 역할: MP4 metadata가 아니라 sampled frames 기준으로 스토리보드, 이미지, 영상의 3자 일치를 평가한다.
  • 구성: agents/video-qa/AGENT.md + scripts/inspect_video.py manifest
  • 사용 예: “5초 validation-run 영상이 교육 의도와 맞는지 검사해줘.”
  • 산출물: continuity, gaze, clarity, storyboard alignment, propose-only 수정안

Skills

작동: trigger 감지 -> SKILL.md와 package reference 읽기 -> topic/storyboard/prompt/QA 산출 -> schema/rubric 검증 -> project 폴더에 저장.

topic-extractor

  • 역할: PPTX/SOP에서 여러 안전교육 주제와 영상 적합도, 우선순위를 뽑는다.
  • 구성: skills/topic-extractor/SKILL.md + source facts + JSON schema
  • 사용 예: uv run python scripts/extract_topics.py --project projects/<slug>
  • 산출물: sources/extracted_topics.json

story-writer

  • 역할: 선택 주제를 위험 인지, 개입, 확인, 교정, 규칙 정리 흐름의 스토리보드로 만든다.
  • 구성: skills/story-writer/SKILL.md + no-narration rule + few-shot examples
  • 사용 예: uv run python scripts/plan_storyboard.py --project projects/<slug> --duration 30
  • 산출물: storyboard/scenes.json, teaching point, subtitle/overlay draft

style-ref-search

  • 역할: 스타일/작업/공간/장비 reference 후보를 모으되 승인 전 prompt 반영은 금지한다.
  • 구성: skills/style-ref-search/SKILL.md + ref/approved/ folder contract
  • 사용 예: uv run python scripts/search_references.py --project projects/<slug> --dry-run
  • 산출물: reference candidates, approved reference metadata, style DNA input

image-consistency-check

  • 역할: 이미지가 스토리, 인물/PPE, 장비, 배경, 스타일, 안전 정확성과 맞는지 점수화한다.
  • 구성: skills/image-consistency-check/SKILL.md + safety_video_harness/image_qa.py
  • 사용 예: uv run python scripts/validate_images.py --project projects/<slug>
  • 산출물: qa/image_reviews.json, qa/visual_review/image_contact_sheet.png, OpenCV MCP drift findings, RALPH critique, regeneration delta, 승인 판단

seedance-prompting

  • 역할: 승인 이미지의 start/end keyframe 기준으로 Seedance motion prompt와 clip plan을 만든다.
  • 구성: skills/seedance-prompting/SKILL.md + Higgsfield local reference + cost gate
  • 사용 예: uv run python scripts/generate_seedance.py --project projects/<slug> --dry-run
  • 산출물: prompts/video_prompts.json, Seedance job plan, 비용 추정 자료

video-inspect

  • 역할: 영상 결과물을 ffprobe metadata와 ffmpeg sampled frames로 검사한다.
  • 구성: skills/video-inspect/SKILL.md + video-frame-analysis/scenelens/understand-video 우선순위
  • 사용 예: uv run python scripts/inspect_video.py --project projects/<slug> --clip video.mp4
  • 산출물: inspection manifest, contact sheet, sampled frames, video QA input

Hooks & Guardrails

작동: PreToolUse에서 live/secret/protected path를 차단하고, PostToolUse에서 schema, scene-link, evidence 검증을 요구한다.

session-start-anchor

  • 역할: no-live, no-TTS, storyboard-first, RALPH, evidence 원칙을 세션 시작에 주입한다.
  • 구성: hooks/session-start-anchor.py
  • 사용 예: 다른 세션에서 프로젝트를 이어받을 때 첫 컨텍스트 앵커로 사용.
  • 기준: 생성보다 인터뷰와 규칙 확인을 먼저 하게 만든다.

pretooluse-live-veto

  • 역할: --live 전 Gate 승인과 외부 업로드 허용 여부를 검사한다.
  • 구성: hooks/pretooluse-live-veto.py + approvals.json + project_config.json
  • 사용 예: generate_images.py --live, generate_seedance.py --live 전 검사.
  • 차단: Gate 미승인 또는 external_upload_allowed=false.

pretooluse-secret-veto

  • 역할: API key, token, bearer secret 같은 문자열이 파일/명령에 들어가는 것을 차단한다.
  • 구성: hooks/pretooluse-secret-veto.py
  • 사용 예: 문서/스크립트 수정 중 secret-like 문자열 검사.
  • 차단: API_KEY|SECRET|TOKEN|Bearer 패턴 발견.

protected-path-veto

  • 역할: AGENTS, hooks, schemas, templates 같은 핵심 경로 변경을 보호한다.
  • 구성: hooks/pretooluse-protected-path-veto.py + hooks/protected_paths.json
  • 사용 예: 플러그인 규칙이나 schema를 바꿀 때 승인/부트스트랩 모드 요구.
  • 기준: 보호 경로 수정은 명시 의도가 있어야 한다.

schema / scene-link

  • 역할: 상태 변경 뒤 schema validation과 sliding-chain scene link 검증을 유도한다.
  • 구성: hooks/posttooluse-schema-validation.py, hooks/posttooluse-scene-link-validation.py
  • 사용 예: 영상 dry-run 전 validate_scene_links.py로 scNN → scNN+1 연결 검사.
  • 차단: start/end keyframe 연결이 깨지면 video work 차단.

stop-sentinel

  • 역할: 작업 완료 선언 전 evidence와 종료 sentinel을 확인한다.
  • 구성: hooks/stop-sentinel-guard.py
  • 사용 예: 장기 작업에서 `.harness/DONE` 없이 종료 선언하지 않도록 방지.
  • 기준: 완료 증거가 없으면 stop 차단.

MCP / External Tooling

작동: plan-only job spec 작성 -> Gate/cost/upload 확인 -> 승인된 경우만 imagegen 또는 Higgsfield 실행 -> project 폴더에 capture -> ffmpeg/QA 검사.

Codex imagegen

  • 역할: 스토리보드와 asset lock contract를 바탕으로 draft 키프레임 이미지를 만든다.
  • 구성: built-in imagegen skill/tool + prompts/image_prompts.json
  • 사용 예: Gate 1 승인 후 asset lock 상태를 확인하고 “sc01부터 draft keyframe을 생성해줘.”
  • 기준: text-only 독립 다중 컷은 production 불가. 이미지는 project images/draft 또는 images/approved에 보존.

Higgsfield CLI / Seedance

  • 역할: 승인된 start/end keyframes와 reference media pack/Soul ID를 영상 clip으로 변환한다.
  • 구성: Higgsfield CLI + seedance_live.py + prompts/video_prompts.json
  • 사용 예: Gate 2 승인 후 5-10초 validation-run을 먼저 실행.
  • 기준: 비용 고지, upload 허용, --execute-paid 없으면 live 실행 금지.

ffmpeg / ffprobe

  • 역할: 영상의 duration/resolution/audio metadata와 sampled frames를 만든다.
  • 구성: 로컬 CLI + video_inspection.py + scripts/inspect_video.py
  • 사용 예: 생성된 mp4에서 start/mid/end/1fps frame 추출.
  • 기준: 영상 QA는 metadata만으로 통과할 수 없고 sampled frame evidence가 필요.

OMO as Runner

  • 역할: 반복 실행자/작업 관리자 역할. 점수 판정자는 아니다.
  • 구성: scripts/plan_omo_image_ralph.py + OMO loop 실행 요청
  • 사용 예: “OMO는 반복 실행자로 쓰고, 판정은 하네스 Arbiter가 하게 해줘.”
  • 기준: 최종 approve/regenerate/escalate 판단은 QA contract와 Arbiter가 담당.

Reference Stores

  • 역할: 사람, 작업, 공간, 장비, PPE, 스타일 참고자료를 분리 저장한다.
  • 구성: model/, product/, ref/approved/, style-guides/
  • 사용 예: 작업상황 사진은 ref/approved/work/, 스타일은 ref/approved/style/.
  • 기준: 승인되지 않은 reference는 prompt에 직접 반영하지 않는다.

Local Docs

  • 역할: 공식문서와 운영 기준을 agent/skill별 참고문서로 로컬화한다.
  • 구성: docs/generative-media-reference-index.md + references/*.md
  • 사용 예: 프롬프트 작성 전 rubric, few-shot, Higgsfield reference를 먼저 확인.
  • 기준: 링크만 두지 않고 프로젝트 판단 기준까지 요약해 둔다.
