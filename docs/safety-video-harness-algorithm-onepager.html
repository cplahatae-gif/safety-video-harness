안전영상 하네스 알고리즘 지도

시작 인터뷰에서 레퍼런스와 목표를 잠그고, 스토리보드를 먼저 만든 뒤, 이미지는 RALPH로 반복 검증하고, 유료 영상은 Gate 2 이후 짧은 검증 영상부터 실행한다. OMO는 반복 실행자이고, 최종 판정은 하네스 평가자와 Arbiter가 맡는다.

실행 순서

01

인터뷰와 자료 등록

교육자료, 목표 주제, 영상 길이, 이미지 분량, 레퍼런스 보유 여부를 먼저 잠근다.

02

주제 후보 추출

PPTX/SOP에서 교육 주제를 뽑고, 사용자가 선택한 주제만 스토리보드 기준으로 쓴다.

03

스토리보드 작성

장면별 교육목표, 출처, 예방행동, 자막, start/end keyframe 계약을 만든다.

04

Gate 1

스토리보드 QA가 통과해야 이미지 live job spec 생성으로 넘어간다.

05

이미지 프롬프트

레퍼런스와 이전 blocker memory를 넣어 Codex imagegen용 작업지시서를 만든다.

06

이미지 RALPH

이미지 QA가 통과할 때까지 최대 20회 반복하고, 같은 blocker 3회면 upstream 수정으로 넘긴다.

07

OMO 실행 관리

OMO는 반복 실행자만 맡고, 점수와 판정은 하네스 role evaluator와 Arbiter가 결정한다.

08

Gate 2와 영상 검증

전체 이미지 QA, 비용, 업로드 허용이 모두 맞을 때 짧은 Seedance 검증 영상만 실행한다.

차단 규칙

Gate 1

qa/storyboard_quality_reviews.json이 통과하지 않으면 스토리보드 승인을 기록하지 않는다.

Gate 2

모든 scene의 이미지 QA coverage, image_qa_loop.passed=true, 승인 이미지, 비용 고지가 필요하다.

Live Veto

--live는 승인 Gate와 external_upload_allowed=true가 모두 있어야 hook에서 허용된다.

Evidence Hygiene

Seedance 결과 URL은 로그에 그대로 저장하지 않고 [redacted-url]로 마스킹한다.

레퍼런스 투입 지도

model/cast

고정 등장인물

작업자, 신호수, 감독자. 얼굴형, 체형, PPE, 포즈 언어를 고정한다.

model/ppe

PPE 기준

안전모, 조끼, 안전화, 장갑. 장면마다 색상과 착용 위치가 흔들리지 않게 한다.

product/equipment

장비·차량

BCT, 덤프트럭, 지게차, 설비. 형태, 축 수, 색상, 스케일을 고정한다.

ref/approved/person

사람 행동 참고

시선, 신호봉, 무전기, 작업자 위치처럼 역할 행동을 참고한다.

ref/approved/work

작업상황 참고

위험구역, 예방행동, 정지/유도/확인 장면의 구도를 참고한다.

ref/approved/space

공간·동선 참고

현장 구조, 차량 동선, 보행자 통로, 사각지대 표시를 참고한다.

ref/approved/style

그림체·톤

안전교육 애니메이션 톤, 색감, 질감, 정보 밀도를 맞춘다.

ref/approved/camera

카메라

와이드샷, 운전석 POV, 후방 확인 장면 등 앵글과 거리감을 참고한다.

ref/approved/lighting

조명·날씨

주간, 흐림, 실내, 공장 조명 등 장면 일관성을 유지한다.
