안전교육 영상 자동제작 하네스 원페이지

PPTX/SOP 교육자료에서 주제를 뽑고, 스토리보드와 키프레임 이미지를 먼저 검증한 뒤, 승인된 경우에만 Higgsfield CLI / Seedance 영상 생성을 실행하는 플러그인형 작업 구조다. 영상은 유료이므로 RALPH loop는 이미지 단계에서 적극적으로 돌리고, 영상 단계에서는 자동 재생성이 아니라 제안만 남긴다. 모든 평가 라운드는 프로젝트 evidence와 llm-wiki/evaluation-rounds.md에 누적된다.

현재 운영 기준

58 pass최근 전체 테스트 기준
30초기본 비용 가드레일
10초유료 검증 실행 제한
3회동일 blocker escalation
비용 잠금 현재 커밋 범위에서는 live imagegen, live Seedance, live TTS, 유료 호출을 모두 금지한다.

1. 전체 파이프라인

스토리보드는 자유롭게 반복 수정하고, 이미지는 점수 기준을 맞출 때까지 재생성하며, 영상은 사용자가 승인한 1회성 짧은 테스트만 실행한다.

자료 등록PPTX/SOP를 프로젝트에 등록하고 원본을 보존한다.
자료 렌더링PPTX 이미지, 슬라이드, 텍스트를 추출한다.
주제 추출교육자료에서 복수 주제를 뽑고 사용자가 선택한다.
레퍼런스 분석인물, 장비, 작업상황, 스타일 DNA를 수동 md 우선으로 읽는다.
스토리보드영상 길이와 이미지 분량에 맞춰 컷과 키프레임을 설계한다.
스토리 QA역할별 평가자를 병렬 실행하고 Arbiter가 통과/수정 여부를 정한다.
Gate 1스토리보드 QA 통과 전 live 이미지 생성 차단.
이미지 생성/QACodex imagegen 작업 명세를 만들고 역할별 평가자와 Arbiter로 판정한다.
이미지 승인승인본은 보존하고 기존 승인 파일은 덮어쓰지 않는다.
Gate 2비용, 업로드 허용, 전체 이미지 QA 통과 확인.
Seedance 생성10초, 최대 1회 검증 실행을 기본으로 제한한다.
영상 QA프레임 검사 후 미달 시 자동 재생성 없이 propose-only 수정 제안만 작성.
승인 게이트 점수 검증/루프 유료 가능 구간

3. 인테이크와 컷 수 알고리즘

영상 시간과 이미지 분량은 사용자에게 먼저 묻는다. 이미지가 많을수록 스토리보드가 촘촘해지고 영상 생성 전 검증할 키프레임이 늘어난다.

질문 1. 몇 초짜리 영상을 만드시겠습니까?
- 기본값: 30초
- 30초 초과는 기술 제한이 아니라 비용/품질 가드레일

질문 2. 이미지의 분량을 어떻게 하시겠습니까?
- 보통: density_factor = 1
- 많이: density_factor = 2
- 더 많이: density_factor = 4

base_segments = ceil(target_duration_sec / 5)
story_beats = base_segments * density_factor
keyframe_count = story_beats + 1

Seedance validation run:
test_seconds = 10
max_attempts = 1
density_factor와 무관하게 짧게 검증

슬라이딩 체인 전제

  • sc01 이미지와 sc02 이미지는 첫 클립의 start/end frame이다.
  • sc02와 sc03은 다음 클립의 start/end frame이다.
  • 따라서 중간 키프레임은 앞 클립의 끝이자 다음 클립의 시작이다.
  • 사람이 갑자기 나타나거나 사라지는 경우는 continuity blocker다.

4. 평가 기준과 RALPH loop 발생 시점

루프는 전부 같은 방식으로 돌지 않는다. 스토리보드는 비용 없이 수정, 이미지는 점수 기준을 맞출 때까지 재생성, 영상은 유료이므로 제안만 남긴다.

단계 평가 항목 통과 기준 미달 시 알고리즘 증거 파일
시나리오/스토리보드 자료 근거, 인과 흐름, 생성 준비도, 교육 전달, Devil's Advocate 리스크 각 항목 4점 이상, critical veto 없음, consensus 통과 무료 수정 루프. 컷을 더 잘게 쪼개거나 원인-예방-결과 구조를 다시 쓴다. qa/storyboard_quality_reviews.json
이미지 기술 조건, 시각 연속성, 스토리 일치, 안전교육성, Devil's Advocate 리스크 필드별 4점 이상, 총점 24점 이상, consensus 통과 RALPH loop 실행. 부족한 지점을 재생성 프롬프트로 기록하고 scNN_vNNN.png 새 버전으로 반복 검증한다. 기준 통과 시 즉시 종료한다. 장면별 최대 20회이며, 같은 blocker가 3회 반복되면 upstream escalation. qa/image_qa_loop.json, qa/arbiter_decisions/image/scNN/round_NNN.json
영상 메타데이터, 프레임 검사, 인물 연속성, 시선 동기, 교육 명확성, Devil's Advocate 리스크 총점 20점 이상, 수동 시각 검토 통과, inspection frame 3개 이상, critical veto 없음 자동 재생성 금지. 비용이 발생하므로 propose_only로 다음 수정 제안만 남기고 사용자 승인 후 재실행한다. qa/video_reviews.json, qa/video_regeneration_proposals.json

5. 유료/외부 호출 가드레일

SessionStart Anchor

  • 새 세션/재개 시 핵심 미션과 금지사항을 재주입한다.
  • no live/paid, no narration/TTS, storyboard-first를 고정한다.
  • 병렬 role evaluator, Arbiter, critical veto, evidence 보존 규칙을 다시 상기시킨다.

이미지 Gate 1

  • 스토리보드 승인 전 --live 차단
  • Codex 내장 imagegen이 기본 경로
  • OpenAI API/CLI 어댑터는 구현하지 않음

영상 Gate 2

  • external_upload_allowed=true 필요
  • --execute-paid 없으면 Seedance live 차단
  • 현재 구현 검증에서는 live/유료 호출을 실행하지 않음

나레이션 제외

  • TTS, voiceover, Whisper/API 전사 제외
  • 교육 문구는 자막, 오버레이, 텍스트 카드로 전달
  • 이미지 안의 생성 텍스트는 기본 금지

6. 역할별 평가와 Arbiter

평가 결과는 다음 생성 라운드의 입력으로 다시 쓰인다. 원본 evidence는 프로젝트 폴더가 source of truth이고, llm-wiki는 반복 실패를 빠르게 읽는 요약 인덱스다.

Role Evaluators

  • qa/role_evaluations/<stage>/<item>/round_NNN.json
  • 각 role은 병렬 실행 가능한 독립 함수로 분리한다.
  • 개별 role의 JSON과 Markdown 발언 파일을 모두 저장한다.

Arbiter

  • qa/arbiter_decisions/<stage>/<item>/round_NNN.json
  • 5/5 승인 또는 4/5+조건부 승인 consensus를 계산한다.
  • critical veto는 다수결보다 우선한다.
  • 같은 signature가 3회 반복되면 이미지 재생성을 멈춘다.

Conditional Debate

  • qa/debates/<stage>/<item>/round_NNN/*.md
  • 점수 불일치, repeated blocker, critical veto 때만 기록한다.
  • Moderator brief, role position, consensus 파일을 남긴다.

7. 라운드 기록 활용

평가 결과는 다음 생성 라운드의 입력으로 다시 쓰인다. 원본 evidence는 프로젝트 폴더가 source of truth이고, llm-wiki는 반복 실패를 빠르게 읽는 요약 인덱스다.

Evidence Bundle

  • qa/evaluation_bundles/<stage>/<item>/round_NNN.json
  • 장면, 리뷰, blocker, 필요한 증거 목록을 묶는다.
  • 평가자는 생성자의 의도 설명 없이 이 bundle만 보고 채점한다.

Round Ledger

  • qa/evaluation_rounds.jsonl
  • 스토리보드, 이미지, 영상 평가 라운드를 append-only로 저장한다.
  • RALPH 현재 횟수, 남은 횟수, repeated blocker signature 판단에 사용한다.

LLM Wiki

  • llm-wiki/evaluation-rounds.md
  • 이전 blocker를 다음 이미지 프롬프트의 Previous QA blockers 섹션에 주입한다.
  • 같은 문제가 반복되면 재생성이 아니라 스토리보드/레퍼런스/프롬프트 계약 수정으로 에스컬레이션한다.

실제 운영에서는 dry-run과 검증 파일을 먼저 남기고, 승인 후에만 live 경로를 연다.

uv run pytest -q

uv run python scripts/generate_images.py \
  --project projects/remicon-collision-guide \
  --dry-run

uv run python scripts/evaluate_storyboard.py --project projects/remicon-collision-guide

uv run python scripts/validate_images.py \
  --project projects/remicon-collision-guide \
  --only sc01

python3 scripts/inspect_video.py \

  --clip projects/remicon-collision-guide/video/clips/sc01_sc02_seedance_validation_10s.mp4 \

  --no-transcript

python3 scripts/validate_video.py \
  --project projects/remicon-collision-guide \
  --expected-clips 1 \
  --clip sc01_sc02_seedance_validation_10s.mp4

9. 품질 실패를 막는 핵심 규칙

스토리

  • 교육자료 출처 없는 안전 주장은 금지한다.
  • 각 장면은 이전 장면의 결과와 다음 장면의 원인을 연결해야 한다.
  • 스토리보드는 영상 생성 전에 충분히 세밀하게 만든다.

이미지

  • 프롬프트에는 인물 특징, PPE, 장비, 작업장 구조, 시선 대상, 이전/다음 장면 연결을 넣는다.
  • 이전 QA blocker는 다음 프롬프트에 명시적으로 들어간다.
  • 승인 이미지는 덮어쓰지 않고 새 버전으로 보존한다.
  • 같은 blocker가 3회 반복되면 이미지 재생성이 아니라 upstream 수정으로 전환한다.

영상

  • MP4 메타데이터만으로는 통과하지 않는다.
  • 샘플 프레임과 OCR evidence를 보고 수동 시각 점수를 기록한다.
  • 시선이 위험원, 신호수, 운전원, 동선 중 무엇을 보는지 설명되어야 한다.
