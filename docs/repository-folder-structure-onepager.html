기존 폴더 이동 감각

이동은 삭제가 아니라 소유권을 명확히 하는 작업이다. 프로젝트 내부는 그대로 둔다.

agents/hooks/skills/.codex-plugin -> app/plugin/

Codex 플러그인 표면을 한 곳에 모은다.

safety_video_harness/scripts/schemas/templates -> app/harness/

실행 엔진과 CLI 내부 구현을 한 버킷으로 묶는다.

style-guides + 공통 cast/ppe/equipment -> references/

여러 프로젝트에서 재사용할 기준만 둔다.

evidence/audio/clips/output/storyboard/scenes.json -> projects/_runs/

루트에 남은 실험 결과와 로그는 manifest와 함께 이동한다.

projects/<slug>/* -> 그대로 유지

스토리보드, 이미지, 영상, QA, 프로젝트 전용 레퍼런스 계약은 유지한다.

마이그레이션 가드레일

CLI 호환 유지 python3 scripts/*.py는 wrapper로 남긴다. 사용자가 호출하는 표면은 깨지지 않는다.
프로젝트 계약 유지 projects/<slug>/model, ref, qa, images는 1차 이동 대상이 아니다.
레퍼런스 경계 references/는 공통 라이브러리다. 프로젝트 승인 refs는 프로젝트 안에 둔다.
Live 금지 유지 구조 변경 중 live imagegen, Seedance, TTS, 외부 업로드, 유료 호출은 실행하지 않는다.
